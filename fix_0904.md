⏺ Bug                                                                                                                                                                
                                                                                                                                                                     
  health.festinalente.dev stuck forever on "Redirecting to your dashboard. If this takes too long, please refresh the page."                                         
                                                                                                                                                                     
  Root cause                                                                                                                                                         
                                                                                                                                                                     
  apps/web/lib/auth-context.tsx created a new Supabase client on every render (const supabase = createClient()) and listed supabase.auth in its effect's dependency
  array. New reference every render → effect tore down and re-ran every render → setLoading(true) fired in a loop → router.replace('/dashboard') got preempted by
  constant re-renders, so navigation never completed.

  Fix (3 files)

  1. apps/web/lib/supabase-browser.ts — Made createClient() a module-level singleton (cached in a let client). Removed the silent demo-URL fallback; now throws if
  NEXT_PUBLIC_SUPABASE_URL / NEXT_PUBLIC_SUPABASE_ANON_KEY are missing.
  2. apps/web/lib/auth-context.tsx — Changed the auth-bootstrap effect's deps from [mounted, supabase.auth] to [] (with eslint-disable comment) so it runs exactly
  once. Removed the now-unnecessary mounted/setMounted gate.
  3. apps/web/app/page.tsx — Added a useRef (hasRedirected) guard around router.replace(target) so it can only fire once, mirroring the pattern in
  apps/web/app/(app)/layout.tsx.
